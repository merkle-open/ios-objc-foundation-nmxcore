How to create namics-cocoapods locally

Current Version of the namics-core-library-ios-objective-c 0.1.0

  1. Checkout pod repo add namics-cocoapods git@git.namics.com:team-mobile/mobile-ios-cocoapods.git

  2. Verify its working cd ~/.cocoapods/repos/namics-cocoapods pod repo lint .

Documentation

Building Documentation

Documentation requires two components:

  • SourceKitten > brew install sourcekitten
  • jazzy > gem install jazzy

Then navigate in terminal to the root path of the repo, where .jazzy.yaml is located and run the following command:

jazzy You will find the updated documentation under ./docs or simply run: open ./docs/index.html

If you added a new section in the Library, make sure to add the related section to the umbrella header file for the documentation generation. See NamicsLibHeaders.h

How to update changes from the library.

Automatic Deployment

You can automatically push your changes to the pod with calling bash deploy.sh It might require administrator rights for creating code documentation automatically, see above

Manual Deployment

  1. Do your changes: Open /Development/NMXLibrary.xcodeproj

  2. Go to the main folder from Terminal, where ‘Namics-Library.podspec’ is located, e.g.: cd ~/Documents/namics-core-library-ios-objective-c/

  3. Validate the pod lib file in terminal: pod lib lint

  4. Commit and push your changes in git, note down the commit-id

  5. Modify the Namics-Library.podspec

  6. spec.version - new version

  7. spec.source - commit to the one from step (5)

  8. Update cocoapod spec: pod repo push namics-cocoapods Namics-Library.podspec –verbose
